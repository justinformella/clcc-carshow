# 8-Bit Homepage (/80s) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an alternate NES/arcade-styled homepage at `/80s` with 8-bit pixel art versions of all image assets.

**Architecture:** New route at `app/80s/` with a layout providing Press Start 2P font and 8-bit CSS. All section components live in `components/8bit/` and replicate the content from the main homepage components with NES-style visuals. A batch API endpoint generates pixel art versions of the hero image, gallery photos, and sponsor logos via the existing Gemini/OpenAI image pipeline.

**Tech Stack:** Next.js App Router, Press Start 2P (Google Fonts), Supabase Storage, Gemini Imagen / OpenAI image generation

### Task 7: 8-Bit Asset Generation API

**Files:**
- Create: `app/api/8bit/generate/route.ts`
- Modify: `types/database.ts`

- [ ] **Step 1: Add `pixel_logo_url` to Sponsor type**

In `types/database.ts`, add `pixel_logo_url: string | null;` to the `Sponsor` type after `logo_url`.

- [ ] **Step 2: Create the batch generation endpoint**

`POST /api/8bit/generate` accepts a JSON body with `{ type: "hero" | "gallery" | "sponsors" | "all" }`.

**Hero generation:**
- Read the hero image from `public/images/PXL_20250831_172232318.jpg` as a buffer
- Upload it to Supabase storage temporarily, get a public URL
- Call the image generation API with a prompt describing an 8-bit pixel art version
- Upload result to `pixel-art/8bit/hero.png`

Actually, the Gemini/OpenAI image APIs take text prompts, not image inputs for style transfer. So for the hero and gallery images, use a descriptive prompt:

**Hero prompt:** `8-bit retro pixel art scene of a charity car show in a downtown small-town American setting. Multiple classic and modern cars parked along tree-lined streets with brick buildings. Crowd of people walking between vehicles. Bright sunny day. Style like a 1990s DOS or NES game screenshot with sharp pixels and authentic retro video game aesthetic.`

**Gallery prompts** (one per image, tailored):
Use descriptive prompts for each curated gallery image. Hardcode 8 prompts:
1. `8-bit pixel art of a classic red Jaguar XK sports car at a car show. Side profile view. NES style.`
2. `8-bit pixel art of a red Ferrari California convertible at a car show. NES style.`
3. `8-bit pixel art of hot rod cars lined up on a small town street. NES style.`
4. `8-bit pixel art of a blue Mazda Miata convertible at a car show. NES style.`
5. `8-bit pixel art of a downtown street filled with classic cars and spectators at a car show. NES style.`
6. `8-bit pixel art of a classic white Corvette C1 at a car show. NES style.`
7. `8-bit pixel art of a convertible Chevy Blazer at an outdoor car show. NES style.`
8. `8-bit pixel art of a blue Shelby GT350 Mustang at a car show. NES style.`

**Sponsor logo generation:**
- Fetch all paid sponsors with `logo_url` set and `pixel_logo_url` not set
- For each, download the logo image, generate prompt: `8-bit retro pixel art version of a company logo. The logo is for [company name]. Sharp pixels, no anti-aliasing, authentic retro video game aesthetic. Black background.`
- Upload result to `pixel-art/8bit/sponsor-{id}.png`
- Update sponsor record with `pixel_logo_url`

Use the existing `generateImage` function from `lib/generate-pixel-art.ts` for the image generation pipeline (Gemini primary, OpenAI fallback).

Return `{ generated, errors }` response.
