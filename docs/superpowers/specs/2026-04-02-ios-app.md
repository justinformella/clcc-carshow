# CLCC Admin iOS/iPad App

**Date:** 2026-04-02
**Status:** Planned

## Overview

A native iOS/iPad app for CLCC event admins to manage registrations, check in vehicles using AI-powered camera identification, and receive real-time push notifications for new registrations.

## Tech Stack

- **Framework:** React Native (Expo)
- **Backend:** Existing Supabase (shared with web app)
- **Auth:** Supabase Auth (same admin accounts)
- **AI Vision:** Gemini / OpenAI Vision API
- **Push Notifications:** Expo Push Notifications
- **Distribution:** TestFlight → App Store
- **Prerequisite:** Apple Developer Account ($99/year)

## Features

### v1: Core App

#### 1. Admin Login
- Supabase email/password auth (same admin credentials as web)
- Persist session with secure storage
- Role-based access (admin, organizer, viewer)

#### 2. Registration List
- Searchable, filterable list of all registrations
- Filter by: payment status, check-in status, award category
- Each row shows: car number, owner name, vehicle year/make/model, payment status, check-in status
- Tap to view full registration detail (same data as web admin detail page)
- Pull-to-refresh

#### 3. AI Camera Check-In
- Tap "Check In" button → opens camera
- Take photo of arriving vehicle
- Photo sent to Gemini Vision / OpenAI Vision API with prompt:
  ```
  Identify this vehicle. Return the year, make, model, and color.
  Respond as JSON: {"year": 2024, "make": "Ford", "model": "Mustang", "color": "Blue"}
  ```
- Fuzzy match AI response against registrations table:
  - Exact match on year + make + model → show as top result with high confidence
  - Partial match (make + model, or just model) → show as lower confidence
  - Color used as tiebreaker
- **Results screen:** Show top 1-3 matching registrations ranked by confidence
  - Each shows: car number, owner name, vehicle details, AI confidence score, check-in status
  - Green highlight if already checked in (prevent double check-in)
  - Tap to confirm → marks registration as checked in via Supabase
  - "None of these" button → manual search fallback
- After check-in confirmation: success haptic feedback + brief confirmation toast

#### 4. Push Notifications
- On first login, register device push token with Supabase (`admin_push_tokens` table)
- When Stripe webhook fires for a new paid registration:
  - Query `admin_push_tokens` table
  - Send push via Expo Push API to all registered admin devices
  - Notification content: "New Registration: {year} {make} {model} — {first_name} {last_name}"
- Tapping notification opens the registration detail in the app

### v2: Speed Check-In Mode

#### 5. Rapid-Fire Check-In
- "Speed Mode" toggle in check-in screen
- After confirming a check-in, camera stays open immediately for the next car
- Running count displayed: "12 of 18 checked in"
- Audio chime on successful check-in
- Queue display showing last 5 check-ins for verification

### v3: Dedicated Check-In Station (iPad)

#### 6. iPad Kiosk Mode
- Landscape-optimized layout for iPad on a stand at the event entrance
- Split view: camera feed on left, registration match results on right
- "Next Car" button to trigger capture (volunteer operates)
- Large, touch-friendly confirm/reject buttons
- Running stats bar: checked in count, remaining, time

#### 7. Continuous Video Mode (Future)
- Live camera feed with motion detection
- When new car detected entering frame (significant visual change):
  - Auto-capture frame
  - Run AI identification
  - Show match candidates
  - Volunteer confirms or dismisses
- Throttled to 1 capture per 3 seconds to manage API costs
- Visual/audio indicator when new car detected

## Database Changes

```sql
-- Push notification tokens for admin devices
CREATE TABLE admin_push_tokens (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  admin_id UUID NOT NULL REFERENCES admins(id),
  token TEXT NOT NULL UNIQUE,
  device_name TEXT,
  platform TEXT DEFAULT 'ios',
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

## API Changes

### Stripe Webhook Enhancement
Add to `app/api/webhooks/stripe/route.ts` `after()` block:
- Query `admin_push_tokens` for all active tokens
- Send Expo push notification with registration details

### New Endpoint: Register Push Token
`POST /api/admin/push-token`
- Body: `{ token: string, device_name: string }`
- Upserts token for the authenticated admin

## Fuzzy Matching Algorithm

```
1. Normalize AI response: lowercase, trim, remove common words ("the", "a")
2. For each registration:
   a. Exact match year + make + model → confidence 100
   b. Make + model match (year off by ≤2) → confidence 85
   c. Make + model match (any year) → confidence 70
   d. Model-only match → confidence 50
   e. Color match as +10 bonus
3. Sort by confidence descending
4. Return top 3 with confidence > 40
```
