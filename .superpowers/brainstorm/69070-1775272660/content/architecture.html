Key Architecture Decision

The bridge: The Phaser game needs to accept a car selection from outside itself.

Right now /arcade has its own Boot → Title → Select flow. For Phase 2, we need the Phaser game to also accept a car ID as input and skip straight to track select → race.

Solution: Build PhaserGame.tsx to accept an optional selectedCar prop. If provided, skip Title/Select and go straight to track selection. If not provided, use its own select flow (standalone /arcade mode).

This means /arcade keeps working standalone during development, and later /race just passes the car in.
